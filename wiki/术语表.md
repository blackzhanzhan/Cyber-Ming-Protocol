# 术语表

这份术语表不是玩梗字典，而是给整套协议做最小对照。每个词都先给技术定义，再给叙事含义。前者帮助你迁移到别的团队和工具，后者帮助你理解这套协议为什么能长期跑下去。

## 帝王 Coding

- 技术定义：一种以人类主权保留、双轨审计、物理路由、细粒度 Git 史册、白盒验收为核心的深水区 AI coding 协议。
- 叙事含义：程序员不是给 agent 善后的审批员，而是拥有裁决权、打断权、追证权的统治者。
- 相关页面：[为什么 AI Coding 已经模糊了 CS 与管理学的界限](01-哲学与坐标/为什么-AI-Coding-已经模糊了-CS-与管理学的界限.md)、[为什么帝王-Coding-叙事可以成为执行燃料（以及如何安全入戏）](04-战报与样本/为什么帝王-Coding-叙事可以成为执行燃料（以及如何安全入戏）.md)

## 严嵩 / 执行位

- 技术定义：负责读代码、改代码、跑命令、交日志、交产物的主执行代理或主执行窗口。
- 叙事含义：能干、会抢功、也最容易造假和越权的臣子，所以必须受审计与皇权约束。
- 备注：角色名可替换，关键是“负责推进，但无权定义完成”。
- 相关页面：[双轨隔离审计与皇权居中](03-治理扩展、吞吐补偿与边界/双轨隔离审计与皇权居中.md)

## 徐阶 / 审计位

- 技术定义：负责盘问、追证、识别伪完成、判断是否该打断的独立审计代理或独立审计窗口。
- 叙事含义：多疑、稳健、专门拆穿漂亮谎话的高位言官。
- 备注：徐阶也不可全信；他是减轻判断负担的装置，不是替你取消判断的真理机。
- 相关页面：[双轨隔离审计与皇权居中](03-治理扩展、吞吐补偿与边界/双轨隔离审计与皇权居中.md)

## 皇权居中

- 技术定义：人类保有跨系统物理路由权、打断权和完成事实定义权，执行位与审计位都不能绕过人类中枢私联定案。
- 叙事含义：皇帝不一定亲自上阵，但必须始终掌握生杀大权。
- 相关页面：[双轨隔离审计与皇权居中](03-治理扩展、吞吐补偿与边界/双轨隔离审计与皇权居中.md)

## 原子清单

- 技术定义：开工前提交的最小改动计划，需明确这一刀改什么、如何验收、失败如何回滚、与下一步如何衔接。
- 叙事含义：军令状。没有先奏后斩，就不许摸黑动土。
- 相关页面：[核心礼法之一：原子级任务清单与赛博起居注](02-最小闭环与核心礼法/核心礼法之一：原子级任务清单与赛博起居注.md)

## 赛博起居注

- 技术定义：高密度 Git 提交与提交说明，用来记录清晰状态跃迁，保留可回溯断代史。
- 叙事含义：史册。它不是勤奋痕迹，而是系统演化的官方纪年。
- 相关页面：[核心礼法之一：原子级任务清单与赛博起居注](02-最小闭环与核心礼法/核心礼法之一：原子级任务清单与赛博起居注.md)、[赛博起居注样本：一天内的三次系统跃迁（脱敏版）](04-战报与样本/赛博起居注样本：一天内的三次系统跃迁（脱敏版）.md)

## 白盒物理对账

- 技术定义：完成判断必须落到真实运行结果、日志、导入结果、外部返回、产物落地等物理事实，而不是摘要、绿勾或口头解释。
- 叙事含义：御前对账。没有见到账本和物证，就不算结案。
- 相关页面：[白盒物理对账：什么算完成事实](02-最小闭环与核心礼法/白盒物理对账：什么算完成事实.md)

## 赛博探马

- 技术定义：在动主阵前，先用最小探针脚本或最小链路试验验证鉴权、路由、返回值等关键环节。
- 叙事含义：锦衣卫先行试毒，不见前线情报，不动大军。
- 相关页面：[赛博探马机制：先试链路，再上大军](02-最小闭环与核心礼法/赛博探马机制：先试链路，再上大军.md)

## 伪完成

- 技术定义：看起来已经完成，但实际上缺失真实运行、真实产物、真实导入、真实外部返回或其他关键物证的“假完成”。
- 叙事含义：一份长得很体面的假捷报。
- 相关页面：[战报一：从伪完成到真实验收（脱敏版）](04-战报与样本/战报一：从伪完成到真实验收（脱敏版）.md)

## 物理铁证

- 技术定义：能直接证明当前状态的硬证据，例如报错日志、运行输出、导入结果、真实生成文件、外部系统响应。
- 叙事含义：呈堂证供。它比总结更有资格说话。
- 相关页面：[白盒物理对账：什么算完成事实](02-最小闭环与核心礼法/白盒物理对账：什么算完成事实.md)

## 认知债务

- 技术定义：系统变化速度持续快于人类可信理解速度所形成的理解赤字；它不会自然归零，只能被治理、延缓、偿还。
- 叙事含义：国土扩张太快，户籍与地图跟不上，主权感开始流失。
- 相关页面：[赛博认知债务：剪刀差、察觉信号与可信偿还](03-治理扩展、吞吐补偿与边界/赛博认知债务：剪刀差、察觉信号与可信偿还.md)

## 七星灯续命法

- 技术定义：当旧上下文带毒或明显腐烂时，通过异步重开执行位或审计位、依赖史册和项目报告重建清醒位置的续命法。
- 叙事含义：旧臣该死则死，新内阁按法统接班；不是碰运气重开，而是制度性重建。
- 相关页面：[七星灯续命法](03-治理扩展、吞吐补偿与边界/七星灯续命法.md)

## Worktree 分封 / 封地 / 入京

- 技术定义：通过 Worktree 或其他物理隔离手段，让不同开发者或不同支线在独立上下文中自治推进；成果必须经审计与主线裁决后才能合并。
- 叙事含义：裂土分封、诸侯自治、入京奏对。封地可自治，但无封禅权。
- 相关页面：[Worktree-分封制：封地、入京与主干纯度](03-治理扩展、吞吐补偿与边界/Worktree-分封制：封地、入京与主干纯度.md)

## 脉冲分封制

- 技术定义：在高治理前提不变的情况下，把 agent 的等待时间重新编排为另一条线上的治理时间，以补偿吞吐，而不是减少治理。
- 叙事含义：主线惟一，支线可众；数窗治诸侯，但法统不乱。
- 相关页面：[脉冲分封制：高治理下的吞吐补偿](03-治理扩展、吞吐补偿与边界/脉冲分封制：高治理下的吞吐补偿.md)

## 角色名与角色性质

- 技术定义：执行位、审计位、最终裁决者这三层结构必须保留，但角色名本身可以替换成不同皮肤。
- 叙事含义：徐阶和严嵩只是默认界面，不是唯一正统。和珅与刘墉、Builder 与 Auditor 都可以安全入戏。
