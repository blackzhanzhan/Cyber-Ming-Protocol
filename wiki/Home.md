# Cyber-Ming-Protocol Wiki

> 当前公开状态：`00`、`01`、`02`、`03`、`04` 与 `术语表` 已开放。

`README.md` 负责项目定义、入口分层与最小上手；Wiki 负责把协议、案例、边界、Skill 接入方式与 Web 审计协作拆成可引用、可深读、可继续扩写的页面体系。

Cyber-Ming-Protocol 也可以被一句话转述为：一个把 AI coding 从黑盒推进改造成审批、执行、审计、续命分层治理的协议项目。

对外最稳妥的记法是：先把项目理解成 `Protocol / Skill / Docs` 三种主落地形态，再把 `web-audit-templates/` 理解成一组分离的 Web 审计协作资产。

如果你是第一次进来，最好的默认动作不是随机跳页，而是先判断自己要走哪条路。

最稳妥的默认顺序是：先过 `00` 前置层，再看一个 30 秒示例，再进入你真正要走的模块正文。

## 第一次进入，先看这里

- 如果你还没分清协议、Skill、Web 审计模板是什么关系，先去 `00-开始这里与落地形态/`
- 如果你已经知道自己只想读理念，就去 `01-哲学与坐标/`
- 如果你要手工跑闭环，就去 `02-最小闭环与核心礼法/`
- 如果你要看治理扩展、续命、分封与边界，就去 `03-治理扩展、吞吐补偿与边界/`
- 如果你要直接看战报、样本与证据页，就去 `04-战报与样本/`
