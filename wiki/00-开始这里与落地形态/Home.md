# 00-开始这里与落地形态

## 这一组页面解决什么问题

如果你是第一次进入 Cyber-Ming-Protocol，最容易混乱的通常不是术语本身，而是项目里现在同时存在多种落地形态：

- 协议本体到底是什么
- `skill/` 里的东西到底算什么
- `web-audit-templates/` 到底要不要安装
- 我到底应该先读、先手工实践，还是先装 Skill

这一组页面就是为了解决这些入口级混乱。

它不负责把所有法统细节重新讲一遍，而负责先把门牌号、路线图与对象关系讲清。

## 先记住最稳妥的总记法

先把项目看成 `Protocol / Skill / Docs` 三种主落地形态，再记住 `web-audit-templates/` 是单独分离出来的 Web 审计协作资产。

在这个总记法下面，再记住最重要的三句话：

1. **Cyber-Ming 首先是一套协议，不是一包 prompt。**
2. **Skill 是强化器，不是协议本体。**
3. **Web 审计模板是协作模板，不是本地 Skill。**

## 四条上手路线

### 路线 A：纯阅读理解

适合还在判断这套协议是否真的值得学的人。

- 从 [`上手路线图：纯阅读、手工实践、Skill 增强、Web 审计`](上手路线图：纯阅读、手工实践、Skill-增强、Web-审计.md) 开始
- 再去 `wiki/01-哲学与坐标/` 与 `wiki/04-战报与样本/`

### 路线 B：无 Skill 手工实践

适合想先不改工具链，只先跑通一次最小闭环的人。

- 先看 [`协议、Skill 与 Web 审计模板：三者关系`](协议、Skill-与-Web-审计模板：三者关系.md)
- 再看 [`上手路线图：纯阅读、手工实践、Skill 增强、Web 审计`](上手路线图：纯阅读、手工实践、Skill-增强、Web-审计.md)
- 想先脑补一轮，再看 [`30 秒最小示例：同一任务的三种跑法`](30-秒最小示例：同一任务的三种跑法.md)
- 然后进入 `wiki/02-最小闭环与核心礼法/`

### 路线 C：安装 Skill 增强执行

适合已经理解协议骨架，想让高频动作更稳定的人。

- 先看 [`什么时候装 Skill，什么时候先别装`](什么时候装-Skill，什么时候先别装.md)
- 再看 [`安装与最小使用`](安装与最小使用.md)
- 想快速对照“没装 / 装了 / Web 复核”三种差别，可先看 [`30 秒最小示例：同一任务的三种跑法`](30-秒最小示例：同一任务的三种跑法.md)
- 最后回到 `wiki/02-最小闭环与核心礼法/` 与 `wiki/03-治理扩展、吞吐补偿与边界/`

### 路线 D：Web 审计协作

适合已经有 IDE 执行位，想把方案审计、完成验收、续命判断交给 Web 侧的人。

- 先看 [`Web 审计模板怎么协作：它不是本地 Skill`](Web-审计模板怎么协作：它不是本地-Skill.md)
- 再对照 `web-audit-templates/` 中的三个模板
