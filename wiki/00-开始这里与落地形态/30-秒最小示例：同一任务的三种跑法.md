# 30 秒最小示例：同一任务的三种跑法

## 这一页解决什么问题

前面的页面已经把协议、Skill、Web 审计模板之间的关系讲清了。

但很多人真正还差最后一步：

**我脑子里到底怎么快速跑一遍？**

这一页就只做这件事。

它不用大任务举例，只用一个足够小、足够普通、能在 30 秒里脑补完一轮的任务来演示三种跑法：

- 不装 Skill，手工跑
- 装了 Skill，更稳地跑
- 需要独立复核时，让 Web 审计位介入

## 示例任务

这次固定用一个很小的任务：

> 修一个设置页保存后不显示成功提示的小 bug。

为什么选这个？因为它足够小，不会把你脑中的最小示例拖成一场大重构；但它又足够真实，仍然能看到方案、执行、验收、复核这几步是怎么接上的。

## 三种跑法总览

| 跑法 | 你先做什么 | 它最能说明什么 | 不是在说明什么 |
| --- | --- | --- | --- |
| 不装 Skill | 先让 IDE 交原子清单 | 协议本身就能手工实践 | 没 Skill 就不能用 |
| 装了 Skill | 用 skill 稳定 planner / executor 节奏 | Skill 是强化器 | Skill 会自动产生完成事实 |
| Web 审计介入 | 把计划或证据包交给 Web 位复核 | 独立审计可以增强稳定性 | 每个任务都必须外置审计 |

## 第一种：不装 Skill，手工跑

### 第一步：先让执行位交方案，不准直接开工

你可以直接对 IDE 说：

```text
我要修一个设置页保存后不显示成功提示的小 bug。

先别直接改代码。
先给我一个尽可能细的原子清单，告诉我准备改哪一层、怎么验收。
```

这一步的目标，不是让 IDE 显得谨慎，而是让它先暴露自己准备怎么做。

如果它能拆成类似下面这种粒度，就已经够你继续了：

- 定位保存成功条件为什么没有触发
- 修复成功提示出现的判断或状态更新
- 补一条最小回归检查，确认保存后确实出现成功提示

### 第二步：再让它施工，但仍然只按证据放行

当方案像样以后，你再让它开始改。

做完后，不要只收一句“已经修好”，而要收这几样：

- 当前运行的测试结果或回归检查结果
- 能证明成功提示真的出现的截图、日志或页面结果
- 对应这一步改动的提交记录

你在这里实践的，其实就是 Cyber-Ming 的最小骨架：

- 先审批
- 再执行
- 最后凭证据验收

到这一步为止，你还完全没有装 Skill，但你已经在使用协议。

## 第二种：装了 Skill，更稳但不替代证据

现在把同一个任务放到已经接入 Skill 的环境里看。

变化不在于任务 magically 完成，而在于高频动作更容易稳定出现。

### planner 更容易先交对的东西

像 `approval-first-planner` 这类 skill，更容易把任务先压成：

- 原子片
- 边界
- 红灯 / 绿灯
- 验收阶梯

这时你更少需要每次手工重复整段长提示。

### executor 更容易守住节奏

像 `approved-checklist-executor` 这类 skill，更容易把后续动作守成：

- 按批准过的片去做
- 做完先验证
- 验完再归档

这会减少“明明知道该这么做，但每次都说不全”的损耗。

### 但最重要的一条完全没变

无论 Skill 触发得多稳定，你最后仍然不能只看格式漂亮的汇报。

你还是要看：

- 当前运行的测试或检查输出
- 成功提示确实出现的证据
- 对应这次状态跃迁的提交记录

也就是说，Skill 让动作更稳，但不替代证据。

## 第三种：需要独立复核时，再让 Web 审计位介入

这一条不是每个小任务都必须上，但当你需要独立复核时，它会非常有价值。

### 什么时候值得让 Web 位介入

最常见的是这几种情况：

- 执行位把这个 bug 说得过于轻松，你怀疑它漏了边界
- 它交上来的是一份很体面的“已完成”，但证据不够硬
- 你想让一个不背 IDE 长上下文的位置独立复核一下

### Web 位看什么

你可以把下面这些材料交给 Web 位：

- 这次任务的原子清单
- 当前修改涉及的关键说明
- 测试结果、截图、日志、提交记录
- 你自己最怀疑的风险点

如果是审方案，优先参考 `web-audit-templates/plan_audit_template.md`。

如果是审完成，优先参考 `web-audit-templates/completion_audit_template.md`。

### Web 位不做什么

真正重要的是它不该做什么：

- 不替你继续写代码
- 不替执行位顺手扩需求
- 不靠总结陈词直接放行

它的职责是独立复核，不是第二个执行位。

## 看完这一页后，你应该立刻能回答的三句话

1. **不装 Skill，也能跑一轮 Cyber-Ming。**
2. **装了 Skill，会让动作更稳，但不会替代证据。**
3. **Web 审计位是在需要独立复核时介入，不是每次都必须先开。**
