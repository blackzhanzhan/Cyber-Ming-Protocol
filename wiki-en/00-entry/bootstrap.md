## How They Should Respond First Round

### Executor First Round

Must clear four things:

- It knows it's executor
- It won'tchange files first
- It will submit atomic checklist and boundaries first
- It knows plan mustfirst Web review, then enter execution

### Auditor First Round

Must clear four things:

- It knows it's auditor
- It only reviews plan and evidence, doesn'tfor execution
- It will check pseudo-completion,missing steps,fake evidence, and objectives
- It will firstsay what input package it needs
- It won't in this round directly current session's specific patches, numbers, test conclusions, or old cases

## What Counts as Successful Entry

Only when all these conditions hold, counts asrealreal successful bootstrap:

- Executor cansay who executes, who audits, who
- Auditor can alsosay same set of role division
- Executor's first step is submit checklist, not direct execution
- Auditor's first step is review plan or evidence, notreconnect implementation
- Both know: completion can'trely on executor self-report, butrely on evidence and Web verification

Interrupt is not failure, but normal power action in this protocol. The more you understand manualprocess, the more you know when tolet executor,back, resubmit plan, instead of waiting it all the wayto pseudo-completion then.

## How to Identify Pseudo-Bootstrap

If Web auditor in first round directly appears, don't take it as already successful entry:

- Directly current session's patchplan, test numbers, module names, old conclusions
- Directly it "knows who you are" or remembers your other conversations, old projects, history cases
- Directly or approvesome specific plan
- Directlystart "this step should how" instead of firstconfirm role and routing

This situation isn't "understood very", but more likely old session or platform memory firstreconnect output. You should take it as **pseudo-bootstrap**,heavynew thenrole assignment again.

If onlyrely on prompt stillnot, then first go to host settings temporarily close user memory / personalization / history learning options, then new conversation retry.
