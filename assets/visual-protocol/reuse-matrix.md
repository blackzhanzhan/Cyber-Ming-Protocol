# Reuse Matrix

## Goal

Every asset should know where it belongs.

This matrix prevents the repo from accumulating nice-looking images that have no clear product job.

## Surface Map

| Asset Type | README | Wiki 00 | Social Post | External Discussion | Notes |
|-----------|--------|---------|-------------|---------------------|-------|
| Trio relationship image | Yes | Yes | Yes | Yes | Best for first memory hit |
| Minimal loop diagram | Yes | Yes | Yes | Yes | Use when explaining the human hub, executor, auditor split |
| Skill vs no Skill diagram | Optional | Yes | Yes | Yes | Best for setup / adoption discussion |
| Contract vs spec / workflow diagram | Optional | Yes | Yes | Yes | Best for methodology positioning |
| Character sheets | No | Optional | Optional | Optional | Mostly internal consistency source |
| Comic storyboards | Optional | Optional | Yes | Yes | Good for posts and threaded explanations |

## Phase-One Recommended Placements

- README:
  - trio relationship image
  - minimal loop diagram
- Wiki layer 1:
  - minimal loop diagram
  - one comic about human sovereignty or execution vs judgment
- Wiki layer 2:
  - Skill vs no Skill diagram
- Wiki layer 3:
  - contract vs spec / workflow diagram
- Social:
  - one image, one judgment only

## Reuse Rules

- Do not redraw the same judgment from scratch for each surface.
- Start from one master `svg` and crop or caption around it if needed.
- Keep graphic text light so the same image works in Chinese and English contexts.
- When longer wording differs by language, put the wording in markdown or social post copy, not inside the image.

## Caption Strategy

- Image carries the structural punch.
- Page copy or post copy carries the full sentence.
- If the judgment cannot be stated in one sentence under the image, the image is too vague.

## Review Questions

Before an asset is considered done, it must answer yes to all of these:

1. Can a first-time repo visitor remember it after five seconds?
2. Does it explain one real governance judgment?
3. Can it be dropped into README, wiki, and a post without redrawing the logic?
4. Does the cast still serve the protocol instead of replacing it?
