# Contract vs Spec / Workflow Diagram Brief

## Core Judgment

Cyber-Ming is not anti-spec and not anti-workflow.
It adds a contract-driven governance layer for the actual slice being executed now.

## Use

- wiki Layer 3 comparison image
- social image for methodology comparison
- external discussion image when people ask how this differs from mainstream approaches

## Information Layers

1. spec: large-direction blueprint
2. workflow: process string and repeated step order
3. contract-driven: current-slice governance object under human routing

## Layout

- three vertical columns
- each column gets one metaphor
- the contract-driven column should feel lighter than spec and harder than ordinary workflow
