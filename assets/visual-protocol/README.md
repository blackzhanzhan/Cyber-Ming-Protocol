# Visual Protocol Layer

This directory is the visual protocol layer for Cyber-Ming.

It does not exist to make the repo look cute.
It exists to make the protocol easier to remember, explain, and circulate.

Phase one serves three goals:

1. Turn the repo from a deep but heavy protocol repo into a memorable, product-like project.
2. Compress complex wiki judgments into diagrams and comic units that are easier to scan and repost.
3. Tie README, wiki, and social surfaces together with one consistent cast and one visual language.

## Phase-One Scope

Phase one is intentionally narrow.

- Three core characters only:
  - Xiao Huanglong
  - Xu Jie
  - Yan Song
- Three core diagrams:
  - minimal loop
  - Skill vs no Skill
  - contract-driven vs spec / workflow
- Four explainer comics:
  - human not stamper
  - execution is not judgment
  - CS vs management
  - contract-driven pulse

## Design Law

- Characters serve structure, not the other way around.
- Every image should carry one judgment.
- Diagrams explain first; comics spread and reinforce second.
- The same asset should work in README, wiki, and social posts with minimal changes.
- Source-of-truth stays auditable: prefer `svg` and `md`.

## Directory Layout

- `style-guide.md`: the shared visual language
- `reuse-matrix.md`: where each asset gets reused
- `characters/`: cast bible, role anchors, character sheets
- `diagrams/`: core governance diagrams and notes
- `comics/`: comic scripts and storyboards

## Naming Rules

- Use lowercase ASCII file names.
- Name by judgment, not by aesthetic treatment.
- Prefer nouns that explain the asset's job:
  - `minimal-loop.svg`
  - `skill-vs-no-skill.svg`
  - `human-not-stamper.svg`
- Keep matching `md` files for the explanatory brief where needed.

## Surface Rules

- README gets the smallest number of highest-memory assets.
- Wiki gets the assets that collapse long explanations into one visual anchor.
- Social posts get single-judgment diagrams or comics that can stand alone out of context.

## Out of Scope for Phase One

- character expansion beyond the core three
- full-color illustration series
- large narrative scenes
- decorative wallpaper art
- platform-specific export bundles

If an asset is beautiful but does not make the governance structure easier to remember, it does not belong here.
